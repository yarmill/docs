Files pulls every attachment a team creates — videos, images, documents — into one
filterable overview, so "that prone-shooting video from camp" is findable months later
without remembering which day it was attached to.

**For:** coaches and athletes · **Where:** Web app

<Info>Files is a **lens over existing attachments**, not a separate store. Every file in
the overview belongs to a day in [Plan](/en/plan/plan) or the
[Training Log](/en/reality/training-log) — there's no way to add a file that isn't
attached to a day.</Info>

## Find a file

Open **Files** for the overview. The header shows **Files • N** — the total file count —
with a breakdown underneath as **N videos · N images · N others**. Pick **Entire Group**
or an individual athlete from the group panel on the left to set what you're looking at.

Below the header sits a single filterable table. Each column header carries a **funnel**
icon — select it to narrow the rows by that column:

<ParamField path="Type">
  The file kind — **video**, **image**, or **other** (documents and anything that isn't a
  video or image).
</ParamField>

<ParamField path="File name">
  The file's name.
</ParamField>

<ParamField path="Tags">
  The labels on the file. A single file can carry several tags at once.
</ParamField>

<ParamField path="Location">
  Where the file lives in the log — the module it's attached to, such as **Plan** or
  **Reality**.
</ParamField>

<ParamField path="Date">
  The date of the file's day in the log.
</ParamField>

When there are no files — or active filters exclude everything — the table shows
**No records**. The counts in the header always reflect the full set, not the filtered
view.

## Why tags, not folders

Files deliberately has **no folder hierarchy**. Folders force one home per file; **tags**
let the same race clip be both *technique* and *prone shooting* at once. Tag consistently
and filtering does the rest — open the **Tags** funnel, narrow to *prone shooting*, and
the whole set surfaces together regardless of which day each clip lives on.

## Add a file

Most files arrive as attachments you make directly on a day in the
[Training Log](/en/reality/training-log) or [Plan](/en/plan/plan). You can also add one
from here — and this entry point can attach **one file to several athletes at once**.

Select **+ Add file** (top right) to open the **Adding file** modal:

<Steps>
  <Step title="Set the date">
    The **Date** field defaults to today — the day the file will attach to.
  </Step>
  <Step title="Choose the module">
    **Module** sets where the file lives in the log, and becomes its **Location** in the
    overview. It defaults to **Plan**.
  </Step>
  <Step title="Add tags">
    Use the **Tags** multi-select to label the file so it's findable later.
  </Step>
  <Step title="Confirm the athletes">
    **Athletes** is prefilled with the current athlete. Add more to attach the same file
    to several athletes at once — useful for one clip that applies to a whole relay squad.
  </Step>
  <Step title="Attach the file and save">
    Use **+ Add file** to attach the upload, then select **Save**. Save stays disabled
    until at least one file is attached; **Cancel** discards everything.
  </Step>
</Steps>

## How it connects

- Files aggregates attachments created on days in the
  [Training Log](/en/reality/training-log) and the [Plan](/en/plan/plan) — these are the
  only sources.
